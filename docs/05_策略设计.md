# 05 策略设计

当前已实现 `core_etf_rotation` 策略、因子计算、ETF 综合评分、目标组合生成、独立风控修正和调仓单生成。

## 默认策略

策略代码：`core_etf_rotation`  
调仓频率：`monthly`

默认因子权重：

```json
{
  "trend": 0.35,
  "momentum": 0.30,
  "volatility": 0.15,
  "drawdown": 0.10,
  "liquidity": 0.05,
  "premium": 0.05
}
```

## 后续实现要求

策略参数必须从 `strategy_config.config` 读取，不允许写死在策略代码中。

策略运行必须创建 `strategy_run`，所有信号、目标仓位、风控结果、调仓单和报告都需要通过 `run_id` 追溯。

综合评分公式：

```text
alpha_score =
trend_score * 0.35
+ momentum_score * 0.30
+ volatility_score * 0.15
+ drawdown_score * 0.10
+ liquidity_score * 0.05
+ premium_score * 0.05
```

## 已实现评分规则

趋势评分：

- 收盘价高于 MA20、MA60、MA120、MA200，每项 25 分。

动量评分：

- 各周期收益率按分段给分，再取平均。
- 收益率 `>= 10%`：100
- 收益率 `>= 3%`：80
- 收益率 `>= 0%`：60
- 收益率 `>= -5%`：40
- 更低：20

波动评分：

- 年化波动越低分数越高。

回撤评分：

- 回撤越小分数越高。

流动性评分：

- 使用当前 ETF 历史 `liquidity_20d` 相对最大值进行 20 到 100 的缩放。

溢价评分：

- 当前版本暂未接入 ETF 净值溢价数据，临时使用中性偏高值 80。后续接入 `etf_nav_premium` 后替换为真实规则。

## 已实现策略运行

策略代码：`core_etf_rotation`

运行流程：

```text
读取 strategy_config
→ 获取指定日期或最新日期 factor_daily 排名
→ 创建 strategy_run
→ 写入 alpha_signal
→ 生成 target_portfolio
```

## 已实现目标组合规则

当前版本先实现 normal 市场的基础目标仓位：

- 第一名权益 ETF：40%
- 第二名权益 ETF：25%
- 防御资产：25%
- 现金资产：10%

如果权益 ETF 不足两只，未使用的权益预算转入现金资产。

防御资产从 `bond` / `gold` 中选择启用且风险等级最低的 ETF。

现金资产从 `cash` 类 ETF 中选择启用 ETF。

当前版本尚未判断 `normal` / `risk_off` / `extreme_risk` 市场状态。策略运行先生成原始目标权重：

```text
raw_target_weight
```

随后风控引擎读取目标仓位并修正：

```text
final_target_weight
```
