```json
{
  "trend": 0.35,
  "momentum": 0.30,
  "volatility": 0.15,
  "drawdown": 0.10,
  "liquidity": 0.05,
  "premium": 0.05
}
```

## 后续实现要求

策略参数必须从 `strategy_config.config` 读取，不允许写死在策略代码中。

策略运行必须创建 `strategy_run`，所有信号、目标仓位、风控结果、调仓单和报告都需要通过 `run_id` 追溯。

综合评分公式：

```text
alpha_score =
trend_score * 0.35
+ momentum_score * 0.30
+ volatility_score * 0.15
+ drawdown_score * 0.10
+ liquidity_score * 0.05
+ premium_score * 0.05
```

## 已实现评分规则

趋势评分：

- 收盘价高于 MA20、MA60、MA120、MA200，每项 25 分。

动量评分：

- 各周期收益率按分段给分，再取平均。
- 收益率 `>= 10%`：100
- 收益率 `>= 3%`：80
- 收益率 `>= 0%`：60
- 收益率 `>= -5%`：40
- 更低：20

波动评分：

- 年化波动越低分数越高。

回撤评分：

- 回撤越小分数越高。

流动性评分：

- 使用当前 ETF 历史 `liquidity_20d` 相对最大值进行 20 到 100 的缩放。

溢价评分：

- 当前版本暂未接入 ETF 净值溢价数据，临时使用中性偏高值 80。后续接入 `etf_nav_premium` 后替换为真实规则。
