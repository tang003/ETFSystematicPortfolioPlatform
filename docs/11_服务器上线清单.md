# 11 服务器上线清单

本文档用于真正部署到云服务器前核对信息。后续把服务器登录方式发给 AI 时，也可以一并提供本文档，方便直接接手。

## 需要你提供的信息

- 服务器公网 IP。
- SSH 登录方式：用户名、端口、密钥或临时密码。
- 域名，以及域名是否已经解析到服务器公网 IP。
- 证书通知邮箱，用于 `ACME_EMAIL`。
- 期望部署目录，例如 `/opt/ETFSystematicPortfolioPlatform`。
- 管理员登录用户名和你希望设置的登录密码。
- PostgreSQL 生产密码，建议使用 24 位以上随机字符串。
- Tushare token；共享 token 建议保留 `TUSHARE_REQUEST_INTERVAL_SECONDS=1.5` 或更高。

## 服务器前置条件

- Ubuntu 22.04 / 24.04 或其他常见 Linux 发行版。
- 已安装 Docker Engine 和 Docker Compose 插件。
- 防火墙或云安全组开放 TCP 22、80、443 和 UDP 443。
- 不要开放 PostgreSQL、Redis、API 或前端内部端口。

## 首次部署流程

```bash
git clone https://github.com/tang003/ETFSystematicPortfolioPlatform.git
cd ETFSystematicPortfolioPlatform
cp .env.production.example .env.production
python3 scripts/generate_auth_secrets.py
```

编辑 `.env.production` 后先做环境校验：

```bash
python3 scripts/validate_production_env.py --env-file .env.production
```

确认无 error 后执行部署：

```bash
python3 scripts/deploy_production.py --env-file .env.production
```

首次部署时 PostgreSQL 还没有运行，脚本会跳过更新前备份；后续更新时会自动先执行数据库备份。

## 后续更新流程

在服务器项目目录执行：

```bash
python3 scripts/deploy_production.py --env-file .env.production
```

脚本会自动执行：

- 检查 `.env.production` 关键配置。
- `git pull --ff-only` 拉取 GitHub 最新代码。
- 如果 PostgreSQL 已运行，先创建数据库备份。
- 校验 `compose.production.yml`。
- 重新构建并启动生产服务。
- 检查 API `/health/ready`。

## 常用排查命令

```bash
docker compose --env-file .env.production -f compose.production.yml ps
docker compose --env-file .env.production -f compose.production.yml logs -f --tail=200 api
docker compose --env-file .env.production -f compose.production.yml logs -f --tail=200 worker
docker compose --env-file .env.production -f compose.production.yml logs -f --tail=200 caddy
```

如果页面打不开：

- 先确认域名解析是否指向服务器公网 IP。
- 再确认安全组开放 80、443 和 UDP 443。
- 查看 `caddy` 日志是否申请证书失败。
- 查看 `api` 和 `worker` 是否 healthy。

如果任务一直 pending：

- 查看 `worker` 是否运行。
- 查看 `worker` 日志中是否有 Tushare 限流、数据源失败或严格数据门禁失败。
- 确认 Redis 正常运行，因为 worker 心跳依赖 Redis。

## 回滚与恢复

每次正式更新前，`deploy_production.py` 会自动调用备份脚本。备份目录为 `backups/`。

恢复示例：

```bash
python3 scripts/restore_database.py backups/quant_etf_YYYYMMDD_HHMMSS.dump \
  --compose-file compose.production.yml \
  --env-file .env.production \
  --confirm-database quant_etf
```

恢复是破坏性操作，只在确认备份正确、需要回滚数据时执行。
