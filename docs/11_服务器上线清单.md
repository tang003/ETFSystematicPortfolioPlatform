# 11 服务器上线清单

## 域名

当前域名：

```text
etf.8886767.xyz
```

由外部 Caddy 反向代理到本项目本机端口。

## 服务器目录

```text
/opt/ETFSystematicPortfolioPlatform
```

## 上线前必须确认

- GitHub 仓库已推送最新代码。
- 服务器可以 `git pull --ff-only`。
- `.env.production` 已配置。
- 数据库密码不是默认值。
- `AUTH_ENABLED=true`。
- 管理员密码哈希已生成。
- `AUTH_SECRET_KEY` 已生成。
- DeepSeek Key 和 Tushare Token 没有写入 Git。
- 服务器上已有其他项目时，不占用 80/443。
- 外部 Caddy 已配置当前域名。

## 部署命令

```bash
cd /opt/ETFSystematicPortfolioPlatform
git pull --ff-only
python3 scripts/deploy_production.py --compose-file compose.production.external-caddy.yml --env-file .env.production
```

## 部署后检查

```bash
docker compose --env-file .env.production -f compose.production.external-caddy.yml ps
curl https://etf.8886767.xyz/health/ready
curl -I https://etf.8886767.xyz/agent-analysis
```

## 数据库检查

```bash
docker compose --env-file .env.production -f compose.production.external-caddy.yml exec -T api python - <<'PY'
from sqlalchemy import inspect
from app.core.database import engine
print("agent_analysis_log" in inspect(engine).get_table_names())
PY
```

## 回滚思路

1. 确认最近一次 Git commit。
2. 如果是代码问题，可以回退到上一 commit 后重新部署。
3. 如果是数据库问题，使用部署前自动备份恢复。
4. 恢复前先保留当前备份，避免二次损坏。

## 安全提醒

- 真实 Key 不进 Git。
- 生产环境不开放 Swagger。
- Cookie 使用 HttpOnly。
- 建议定期轮换 DeepSeek、Tushare、数据库密码。
- 如果 Key 曾在聊天工具或截图中出现，正式运营前应重新生成。
