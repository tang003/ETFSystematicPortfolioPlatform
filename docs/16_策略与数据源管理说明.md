# 16 策略与数据源管理说明

本文说明本系统如何管理 ETF 策略和外部数据接口。以后新增策略、切换数据源、排查 Token 配置时，优先阅读本文。

## 1. 策略管理

### 当前内置策略

系统会自动补齐以下内置策略配置：

| 策略代码 | 策略名称 | 风险画像 | 说明 |
| --- | --- | --- | --- |
| `core_etf_rotation` | 核心 ETF 轮动策略 | balanced | 默认策略，权益、防守资产和现金较均衡 |
| `aggressive_equity_rotation` | 进取权益轮动策略 | aggressive | 提高权益 ETF 权重，更偏趋势捕捉 |
| `defensive_all_weather` | 防守全天候配置策略 | defensive | 提高债券、黄金和现金缓冲，优先控制回撤 |
| `global_qdii_rotation` | QDII 全球 ETF 轮动策略 | global | 优先在跨境和全球 ETF 中筛选 |

### 策略配置表

策略配置保存在：

```text
strategy_config
```

核心字段：

| 字段 | 含义 |
| --- | --- |
| `strategy_code` | 策略唯一代码，前端运行策略时传这个值 |
| `strategy_name` | 策略显示名称 |
| `version` | 策略版本 |
| `rebalance_frequency` | 调仓频率，例如 monthly、quarterly、manual |
| `config` | 策略参数 JSON |
| `enabled` | 是否启用 |

### 当前策略引擎

当前正式接入的策略引擎是：

```text
factor_rotation
```

它基于 `factor_daily.alpha_score` 排名选择 ETF，再结合可交易性评分和目标权重生成 `target_portfolio`。

不同策略主要通过 `config` 参数产生差异：

```json
{
  "engine": "factor_rotation",
  "risk_profile": "balanced",
  "target_weights": {
    "equity_primary": 0.4,
    "equity_secondary": 0.25,
    "defense": 0.25,
    "cash": 0.1
  },
  "preferred_regions": ["US", "GLOBAL"]
}
```

参数说明：

| 参数 | 含义 |
| --- | --- |
| `equity_primary` | Alpha 排名第一的权益 ETF 目标权重 |
| `equity_secondary` | Alpha 排名第二的权益 ETF 目标权重 |
| `defense` | 债券、黄金等防守资产目标权重 |
| `cash` | 现金或货币 ETF 缓冲权重 |
| `preferred_regions` | 可选；限制策略优先选择的区域 |

如果权重合计不是 100%，后端会自动归一化。

### 如何新增策略

普通参数型策略：

1. 打开前端“策略管理”。
2. 点击“新增策略”。
3. 输入策略代码、名称、目标权重、风险画像和说明。
4. 保存后启用。
5. 在“全流程”或“目标组合”页面选择该策略运行。

复杂算法型策略：

1. 先在 `strategy_config` 中增加策略配置。
2. 在 `backend/app/services/strategy_service.py` 中增加新的策略引擎或分发逻辑。
3. 让 `run_strategy` 根据 `strategy_code` 或 `config.engine` 调用对应算法。
4. 输出仍然统一写入 `strategy_run`、`alpha_signal`、`target_portfolio`。
5. 更新本文档和 `docs/09_开发日志.md`。

## 2. 数据源管理

### 当前正式数据源

当前正式行情和 ETF 主资料只使用：

```text
Tushare Pro
```

原因：

- 避免 AKShare、东方财富、Tushare 字段口径混用。
- 便于追踪行情来源。
- 便于控制请求频率和 Token 使用。

AI 投研解释使用：

```text
DeepSeek
```

### 前端管理页

前端新增：

```text
/data-sources
```

显示内容：

- 默认日历源。
- 默认行情源。
- 默认资料源。
- 默认 AI 源。
- Token 是否配置。
- Token 脱敏显示。
- API 地址。
- 请求间隔。
- 每分钟 / 每日限额。
- 业务适配状态。
- 估算每分钟频率。
- 各数据源被哪些模块使用。

可操作内容：

- 新增数据源。
- 编辑数据源名称、类型、API 地址、鉴权方式、限频和用途。
- 写入或替换密钥。
- 清空数据库中保存的密钥。
- 启用或停用数据源。

注意：新增数据源默认只是“配置登记”。如果要真正用于 ETF 行情、交易日历、ETF 主资料或新闻同步，还需要后端为该数据源开发适配器，包括字段映射、分页、限频、错误处理和清洗规则。

### 为什么不能在前端保存 Token

Token 不应该放在浏览器前端。

原因：

- 浏览器 JS 可以被用户查看。
- 网络请求可以被抓包。
- 浏览器插件可能读取页面数据。
- 一旦泄漏，别人可以消耗你的接口额度。

正确做法：

```text
Token 保存在服务器 .env.production 或后端数据源配置表中。
前端只显示是否配置、脱敏 Token 和健康状态。
```

当前实现中，前端只负责提交新密钥；接口返回时永远只返回脱敏值，不返回明文。

### 当前环境变量

Tushare：

```text
TUSHARE_TOKEN
TUSHARE_API_URL
TUSHARE_REQUEST_INTERVAL_SECONDS
```

DeepSeek：

```text
DEEPSEEK_API_KEY
DEEPSEEK_BASE_URL
DEEPSEEK_MODEL
DEEPSEEK_TIMEOUT_SECONDS
DEEPSEEK_MAX_TOKENS
```

### 数据源配置表

当前已经新增数据库表：

```text
data_source_config
```

主要字段：

| 字段 | 含义 |
| --- | --- |
| `provider_code` | 数据源代码 |
| `provider_name` | 数据源名称 |
| `provider_type` | 数据源类型，例如 market、ai、news、broker |
| `enabled` | 是否启用 |
| `base_url` | API 地址 |
| `auth_type` | 鉴权方式，例如 token、bearer、none |
| `secret_value` | 后端保存的密钥，前端不回显 |
| `request_interval_seconds` | 请求间隔 |
| `quota_per_minute` | 每分钟限额 |
| `quota_per_day` | 每日限额 |
| `supported_usages` | 支持用途，例如 market_daily、asset_profile、ai_research |
| `adapter_status` | 业务适配状态 |
| `notes` | 备注说明 |

`adapter_status` 含义：

| 状态 | 含义 |
| --- | --- |
| `runtime_supported` | 已接入运行时，可以被对应业务服务读取 |
| `metadata_only` | 只是登记配置，还不能直接用于同步 |
| `planned` | 计划接入或开发中 |

### 当前可直接运行时覆盖的源

目前已经支持从数据库配置覆盖运行时参数：

| 数据源 | 可覆盖内容 | 影响模块 |
| --- | --- | --- |
| `tushare` | `base_url`、`secret_value` | Tushare trade_cal、fund_daily、fund_basic、fund_nav、fund_share、index_daily |
| `deepseek` | `base_url`、`secret_value` | AI 投研和报告增强 |

如果数据库中没有配置密钥，系统会继续使用服务器环境变量。

## 3. API

策略：

- `GET /api/strategies`
- `POST /api/strategies`
- `PATCH /api/strategies/{strategy_code}`
- `POST /api/strategies/run`

数据源：

- `GET /api/settings/data-sources`
- `POST /api/settings/data-sources`
- `PATCH /api/settings/data-sources/{provider_code}`

## 4. 使用建议

普通用户：

1. 不需要修改数据源。
2. 默认使用核心 ETF 轮动策略。
3. 想看更高收益弹性时，用进取权益策略试算。
4. 想控制回撤时，用防守全天候策略试算。
5. 任何策略都必须先保证 ETF 有足够历史行情和因子数据。

高级用户：

1. 先在策略管理中新建参数型策略。
2. 用回测模块观察收益、回撤、胜率和换手。
3. 再进入全流程生成目标组合和报告。
4. 不建议直接用新策略指导真实资金，至少需要多周期回测和人工复核。
