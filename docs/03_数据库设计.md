# 03 数据库设计

数据库：PostgreSQL

默认库名：`quant_etf`

ORM：SQLAlchemy

迁移工具：Alembic

## 迁移约定

- 初始 SQL 位于 `sql/init_schema.sql` 和 `sql/seed_assets.sql`，只负责首次创建 volume 时的初始化。
- 代码版本更新后的表结构变化必须使用 Alembic。
- API 和 worker 容器启动时会先执行 `alembic upgrade head`。
- 新增表、字段、索引时必须同步更新本文档和 `docs/09_开发日志.md`。

## 核心表

### asset_master

用途：ETF 主数据和研究池。

关键字段：

- `symbol`：ETF 代码，唯一。
- `name`：名称。
- `exchange`：交易所，例如 `SH`、`SZ`。
- `asset_class`：资产类别，例如 `equity`、`bond`、`gold`、`cash`、`qdii`。
- `asset_region`：区域，例如 `CN`、`HK`、`US`、`GLOBAL`。
- `currency`：币种，默认 `CNY`。
- `is_cross_border`：是否跨境 ETF。
- `is_leveraged`：是否杠杆。
- `is_inverse`：是否反向。
- `enabled`：是否进入研究池。
- `risk_level`：风险等级，1 最低，5 最高。
- `fund_company`：基金公司。
- `tracking_index`：跟踪指数。
- `listing_date`：上市日期。
- `fund_size`：基金规模，单位为元。
- `management_fee`：管理费率，小数格式，例如 0.005 表示 0.5%。
- `custody_fee`：托管费率。
- `expense_ratio`：综合费率。
- `tracking_error`：跟踪误差。
- `latest_premium_rate`：最新溢价率。
- `description`：说明。

说明：

- 全市场基础池可以通过东方财富分页源同步；当外部分页源不稳定时，`source=auto` 会使用 Tushare `fund_basic` 作为兜底，再尝试 AKShare。
- 新同步 ETF 默认不启用研究，避免全市场都进入策略。
- 当前持仓里输入新代码时，系统可以自动登记候选 ETF。

### trading_calendar

用途：交易日历。

关键字段：

- `trade_date`
- `market`
- `is_open`

用于判断行情缺失、同步范围和调仓日期。

### market_data_raw

用途：保存外部数据源原始行情。

唯一约束：`symbol + trade_date + source`

关键字段：

- `symbol`
- `trade_date`
- `open`
- `high`
- `low`
- `close`
- `volume`
- `amount`
- `source`
- `raw_payload`

### market_data_clean

用途：保存清洗后的行情，是策略、因子、回测、ETF 详情的主要行情来源。

唯一约束：`symbol + trade_date`

关键字段：

- `symbol`
- `trade_date`
- `open`
- `high`
- `low`
- `close`
- `volume`
- `amount`
- `is_adjusted`
- `data_status`

### etf_nav_premium

用途：保存 ETF 净值、市场价和溢价率。

当前是预留和部分能力，后续做跨境 ETF、LOF、QDII 风控时会更重要。

### data_quality_log

用途：记录数据同步、清洗、质量检查的结果。

常见类型：

- `missing_data`
- `duplicate_data`
- `zero_price`
- `zero_volume`
- `abnormal_return`
- `premium_missing`
- `cleaning_success`
- `market_sync_failed`

### factor_daily

用途：保存每日因子和 Alpha 综合评分。

关键字段：

- 均线：`ma20`、`ma60`、`ma120`、`ma200`
- 收益：`ret_20d`、`ret_60d`、`ret_120d`、`ret_250d`
- 波动：`volatility_60d`、`volatility_120d`
- 回撤：`drawdown_120d`、`drawdown_250d`
- 流动性：`liquidity_20d`
- 溢价：`premium_rate`
- 评分：`trend_score`、`momentum_score`、`volatility_score`、`drawdown_score`、`liquidity_score`、`premium_score`、`alpha_score`

### strategy_config

用途：保存策略参数。

当前核心策略：`core_etf_rotation`

### strategy_run

用途：记录每次策略运行。

策略结果都应能追溯到 `run_id`。

### alpha_signal

用途：保存策略生成的 ETF 排名和信号。

关联：`run_id -> strategy_run.id`

### target_portfolio

用途：保存策略生成的目标组合权重。

关键字段：

- `run_id`
- `portfolio_date`
- `symbol`
- `raw_target_weight`
- `final_target_weight`
- `asset_class`
- `reason`

### portfolio_position

用途：保存用户当前持仓快照。

唯一约束：`position_date + symbol`

关键字段：

- `position_date`：持仓日期。
- `symbol`：ETF 代码。
- `position_name`：持仓名称，系统按代码补全。
- `asset_type`：资产类型，通常是 `etf`。
- `quantity`：持仓数量。
- `current_price`：现价，系统从行情补全。
- `cost_price`：成本价，用户填写。
- `market_value`：市值，系统计算。
- `weight`：组合权重，系统计算。
- `cost_basis`：持仓成本，系统计算。
- `unrealized_pnl`：浮盈亏，系统计算。
- `unrealized_pnl_rate`：浮盈亏率，系统计算。

### holding_analysis_result

用途：保存当前持仓与目标组合的差异分析。

关键字段：

- `run_id`
- `analysis_date`
- `symbol`
- `current_weight`
- `target_weight`
- `weight_diff`
- `action_suggestion`：`ADD`、`REDUCE`、`HOLD`、`REDUCE_OR_EXIT`
- `alpha_score`
- `reason`

### investment_plan

用途：保存定投计划。

关键字段：

- `plan_name`
- `run_id`
- `start_date`
- `months`
- `monthly_amount`
- `total_budget`
- `target_annual_return`
- `investment_mode`
- `status`
- `note`

说明：目标年化只是策略参数，不是收益承诺。

### investment_plan_suggestion

用途：保存每期定投建议。

关键字段：

- `plan_id`
- `run_id`
- `suggestion_date`
- `period_no`
- `symbol`
- `target_weight`
- `current_weight`
- `gap_weight`
- `suggested_amount`
- `action_suggestion`
- `reason`

### risk_rule

用途：保存风控规则配置。

### risk_check_result

用途：保存风控检查结果。

### rebalance_order

用途：保存调仓建议单。

当前只是建议单，不会自动下单。

### backtest_run / backtest_equity_curve / backtest_trade / backtest_metrics

用途：保存回测运行、净值曲线、模拟交易记录和指标。

### report_log

用途：保存 Markdown 报告。

### workflow_task / workflow_task_step

用途：保存全流程任务和每一步状态。

生产环境由 worker 执行 pending 任务。

### agent_analysis_log

用途：保存 AI 投研委员会历史记录。

新增版本：`v0.39.0-agent-analysis-history`

关键字段：

- `symbol`：ETF 代码。
- `name`：ETF 名称。
- `start_date` / `end_date`：分析区间。
- `data_status`：数据状态，例如 `ready` 或 `missing_market_data`。
- `llm_used`：是否实际使用 DeepSeek。
- `llm_model`：模型名。
- `final_action`：最终建议。
- `final_score`：综合评分。
- `final_summary`：最终总结。
- `manager_commentary`：复合经理说明。
- `agents_payload`：各 Agent 的观点、证据、风险、建议。
- `warnings_payload`：数据缺失或限制提示。
- `created_at`：生成时间。

说明：

- 不保存 DeepSeek API Key。
- 每次运行 AI 投研都会新增一条记录。
- 前端 AI 投研页会读取最近历史记录。

```text
  -> alpha_signal
  -> target_portfolio

portfolio_position
  -> holding_analysis_result
  -> rebalance_order

investment_plan
  -> investment_plan_suggestion

strategy_run
  -> alpha_signal
  -> target_portfolio
  -> risk_check_result
  -> rebalance_order

workflow_task
  -> workflow_task_step

agent_analysis_log
  -> 独立保存 AI 投研快照
```
