# 03 数据库设计

数据库：PostgreSQL  
数据库名：`quant_etf`  
初始化文件：`sql/init_schema.sql`  
种子数据：`sql/seed_assets.sql`
迁移工具：Alembic，配置位于 `backend/alembic.ini` 和 `backend/alembic/`

## 迁移约定

- `sql/init_schema.sql` 和 `sql/seed_assets.sql` 只负责 PostgreSQL volume 首次创建时的初始化。
- v0.22.0 起，代码版本更新后的结构变化由 Alembic 管理。
- API 容器启动命令会先执行 `alembic upgrade head`，迁移成功后才启动 FastAPI。
- 当前基线版本为 `20260710_0001`，对应已有 SQLAlchemy ORM 模型。
- 后续新增表、字段、索引时，应新增 Alembic revision，并同步更新本文档和开发日志。

## 表清单

### asset_master

用途：ETF 主数据池，管理系统可研究、可交易的 ETF。

主键：`id`
唯一约束：`symbol`

关键字段：

- `symbol`：ETF 代码。
- `name`：ETF 名称。
- `exchange`：交易所，例如 SH、SZ。
- `asset_class`：资产类别，例如 equity、bond、gold、cash。
- `asset_region`：资产区域，例如 CN、US、GLOBAL。
- `is_cross_border`：是否跨境 ETF。
- `enabled`：是否纳入系统。
- `risk_level`：风险等级，1 最低，5 最高。

说明：

- 当前初始化 SQL 提供的是“起步样本池 + 扩展示例池”，不是全市场 ETF 全量清单。
- 后续可以通过前端“ETF 池”页面批量导入自己的 ETF 主数据，不需要修改 SQL。

### market_data_raw

用途：保存外部数据源拉取的原始行情，不直接用于策略。

主键：`id`
唯一约束：`symbol, trade_date, source`

### market_data_clean

用途：保存清洗后的行情，用于因子、策略和回测。

主键：`id`  
唯一约束：`symbol, trade_date`

### etf_nav_premium

用途：保存 ETF 净值、市场价格和溢价率，尤其用于跨境 ETF 风控。

主键：`id`  
唯一约束：`symbol, trade_date`

计算口径：

```text
premium_rate = (market_price - nav) / nav
```

### trading_calendar

用途：交易日历，用于判断缺失数据和调仓日期。

主键：`id`  
唯一约束：`trade_date`

### data_quality_log

用途：记录数据缺失、重复、异常、清洗结果。

主键：`id`

常见 `check_type`：

- `missing_data`
- `duplicate_data`
- `zero_price`
- `zero_volume`
- `abnormal_return`
- `premium_missing`
- `cleaning_success`

### factor_daily

用途：保存每个 ETF 每个交易日的因子和评分。

主键：`id`  
唯一约束：`symbol, trade_date`

因子包括：

- 均线：`ma20`、`ma60`、`ma120`、`ma200`
- 收益：`ret_20d`、`ret_60d`、`ret_120d`、`ret_250d`
- 波动：`volatility_60d`、`volatility_120d`
- 回撤：`drawdown_120d`、`drawdown_250d`
- 流动性：`liquidity_20d`
- 溢价：`premium_rate`
- 评分：趋势、动量、波动、回撤、流动性、溢价、综合 alpha

### strategy_config

用途：保存策略参数，避免把参数写死在代码中。

主键：`id`  
唯一约束：`strategy_code`

当前种子策略：`core_etf_rotation`

### strategy_run

用途：记录每次策略运行。

主键：`id`

要求：所有策略结果必须可追溯到 `run_id`。

### alpha_signal

用途：记录策略模型生成的 ETF 排名和评分。

主键：`id`  
外键：`run_id → strategy_run.id`

### target_portfolio

用途：保存风控前和风控后的目标仓位。

主键：`id`  
外键：`run_id → strategy_run.id`

### portfolio_position

用途：记录用户当前持仓，用于生成调仓建议。

主键：`id`  
唯一约束：`position_date, symbol`

关键字段：

- `position_date`：持仓快照日期。
- `symbol`：证券代码，可为 ETF、场内基金、股票或其他资产。
- `position_name`：持仓名称，例如中概互联、消费 ETF。
- `asset_type`：资产类型，例如 `etf`、`stock`、`cash`、`other`。
- `quantity`：持有份额，可为空。
- `current_price`：当前价格，对应券商“现价”。
- `cost_price`：成本价格，对应券商“成本”。
- `market_value`：当前市值。
- `weight`：按本次快照总市值计算出的当前权重。
- `cost_basis`：持仓成本，可为空。
- `unrealized_pnl`：持仓盈亏。
- `unrealized_pnl_rate`：持仓收益率。

### holding_analysis_result

用途：保存当前持仓与目标组合的差异分析，用于判断加仓、减仓、持有或退出。

主键：`id`
外键：`run_id → strategy_run.id`

关键字段：

- `analysis_date`：分析日期。
- `symbol`：ETF 代码。
- `current_weight`：当前持仓权重，来自 `portfolio_position`。
- `target_weight`：目标组合权重，来自 `target_portfolio.final_target_weight`。
- `weight_diff`：目标权重减当前权重。
- `action_suggestion`：动作建议，取值包括 `ADD`、`REDUCE`、`HOLD`、`REDUCE_OR_EXIT`。
- `alpha_score`：最近一次因子评分，用于辅助解释。
- `reason`：中文分析原因。

### investment_plan

用途：保存用户定投计划，只做分析建议，不触发真实扣款或交易。

主键：`id`
外键：`run_id → strategy_run.id`

关键字段：

- `plan_name`：计划名称。
- `run_id`：使用哪次策略目标组合进行分析，可为空；为空时使用最近目标组合。
- `start_date`：计划开始日期。
- `months`：定投期数。
- `monthly_amount`：每期投入金额。
- `total_budget`：计划总预算，等于 `months * monthly_amount`。
- `status`：计划状态，当前默认 `active`。
- `note`：备注。

### investment_plan_suggestion

用途：保存某个定投计划某一期的投入建议。

主键：`id`
外键：`plan_id → investment_plan.id`，`run_id → strategy_run.id`

关键字段：

- `suggestion_date`：建议生成日期。
- `period_no`：第几期定投。
- `symbol`：建议投入的 ETF 代码。
- `target_weight`：目标权重。
- `current_weight`：当前持仓权重。
- `gap_weight`：目标权重减当前权重。
- `suggested_amount`：本期建议投入金额。
- `action_suggestion`：动作建议，当前为 `INVEST`。
- `reason`：中文解释。

### risk_rule

用途：保存风控规则。

主键：`id`  
唯一约束：`rule_code`

当前种子规则：

- `max_single_weight`
- `max_equity_weight`
- `min_defense_weight`
- `min_cash_weight`
- `premium_buy_limit`

### risk_check_result

用途：记录每次风控检查结果。

主键：`id`  
外键：`run_id → strategy_run.id`

### rebalance_order

用途：根据当前持仓和目标仓位生成调仓建议，不连接真实交易。

主键：`id`  
外键：`run_id → strategy_run.id`

### backtest_run

用途：记录回测任务。

主键：`id`

### backtest_equity_curve

用途：记录回测净值曲线。

主键：`id`  
外键：`backtest_id → backtest_run.id`

### backtest_trade

用途：记录回测交易明细。

主键：`id`  
外键：`backtest_id → backtest_run.id`

### backtest_metrics

用途：记录回测绩效指标。

主键：`id`  
外键：`backtest_id → backtest_run.id`

### performance_attribution

用途：记录收益归因结果。

主键：`id`  
外键：`run_id → strategy_run.id`

### report_log

用途：记录月度报告、归因报告等 Markdown 报告。

主键：`id`  
外键：`run_id → strategy_run.id`

## 初始化方式

Docker Compose 首次启动 PostgreSQL 时会自动执行：

```text
sql/init_schema.sql
sql/seed_assets.sql
```

如果已有 `postgres_data` volume，修改 SQL 后不会自动重新执行。开发阶段可以删除 volume 后重建：

```bash
docker compose down -v
docker compose up --build
```
