# 01 项目说明

## 项目定位

项目名称：ETF Systematic Portfolio Platform  
中文名称：ETF 系统化资产配置平台

本项目面向个人中长期 ETF 投研与资产配置，目标是把“数据、策略、风控、持仓、调仓、回测、报告”沉淀为可追踪、可复盘、可解释的工程流程。

本系统不是短线预测系统，不是荐股系统，也不是自动实盘交易系统。当前所有交易相关输出都只是建议单，不会连接券商账户，也不会自动操作资金。

## 核心原则

- 数据可信先于回测。
- 风控先于收益。
- 人工确认先于自动交易。
- ETF 优先于个股。
- 中长期配置优先于短线交易。
- 可解释规则优先于黑箱模型。
- 文档与代码同步更新，保证后续可以交给自己或 AI 快速接手。

## 当前版本范围

当前版本：`v0.37.1-etf-detail-sync-hint`

已实现能力：

- FastAPI 后端项目结构。
- SQLAlchemy 数据库连接和核心 ORM。
- PostgreSQL 初始化 SQL 和 ETF 种子资产池。
- Docker Compose 编排 PostgreSQL、Redis、API 和 Web 控制台。
- ETF 主数据查询、手动导入和全市场基础池同步。
- 交易日历同步。
- ETF 行情同步，AKShare 失败时 fallback 到东方财富 K 线接口。
- 行情清洗和数据质量检查。
- Alpha 因子计算和排名。
- 因子研究实验室，支持 IC、Rank IC、因子相关性和分组未来收益。
- `core_etf_rotation` 策略运行。
- 目标组合生成。
- 独立风控检查。
- 当前持仓快照录入。
- 当前持仓与目标组合差异分析。
- 组合工作台集中展示目标组合、当前持仓、定投建议和调仓建议。
- 定投计划创建。
- 基于目标组合和当前仓位缺口生成本期定投建议。
- 基于真实当前权重生成调仓建议单。
- 基础等权买入持有回测。
- 策略月度调仓回测，支持月度追加资金、手续费和滑点。
- 月度 Markdown 报告。
- Vue 3 + Element Plus 中文前端控制台。
- 前端路由懒加载和构建分包，降低首屏一次性加载体积。
- 全流程后台任务、任务日志、取消任务、失败步骤重试。
- 系统状态看板：集中查看 API 存活、就绪状态、数据质量摘要和最近任务异常。
- 管理员登录保护、HttpOnly Cookie 会话和生产环境鉴权。
- 前端 Nginx 基础安全响应头，包括 `Content-Security-Policy`、`X-Frame-Options`、`X-Content-Type-Options`、`Referrer-Policy` 和 `Permissions-Policy`。
- 生产代理登录限流加固：启用 `TRUST_PROXY_HEADERS` 时按右侧代理追加 IP 识别客户端，降低伪造 `X-Forwarded-For` 首段绕过限流的风险。
- 生产部署脚本、数据库迁移、备份恢复和外部 Caddy 接入。
- 当前持仓弹窗维护：新增、编辑和补仓都通过弹窗完成，用户只填写代码、持仓数量和成本价，系统补全名称、类型和现价。
- ETF 池筛选器：支持按类别、区域、风险上限、跨境状态和启用状态维护研究 ETF 范围。
- ETF 全市场基础池同步：通过 AKShare/东方财富 ETF 列表导入全市场 ETF 主数据，默认不启用研究，避免策略池被一次性放大。
- ETF 对比页：支持多只 ETF 标准化净值、收益、波动、回撤、成交额、相关性和可交易性评分对比。
- ETF 池可交易性评分：ETF 池列表展示评分，支持按评分下限筛选；工作流市场门禁结果会附带低评分 warning。
- 因子和策略可交易性提示：因子排名页展示评分，策略目标组合 reason 中记录入选 ETF 的可交易性状态。
- 策略可交易性约束：权益 ETF 可交易性评分低于 40 时剔除，40-60 时降权到半仓，释放权重转入现金缓冲。
- ETF 详情页：支持查看单只 ETF 的基础画像、净值与回撤曲线、可交易性、最新因子和最近行情。
- ETF 详情页行情缺失时会明确提示，并支持一键同步当前 ETF 的本地行情后刷新。
- 组合风险透视：按资产类别、区域和风险等级查看当前持仓与目标组合的暴露差异。
- 目标定投计划：支持准备资金、投入期限、目标年化收益和执行模式参数，支持固定节奏、回撤增强、信号择时和未来自动执行预览，当前只生成建议，不自动下单。
- 月度报告增强：报告包含 ETF 池状态、当前持仓快照、定投建议、调仓建议和回测摘要。

暂未实现能力：

- 真实券商账户连接。
- 自动下单。
- 多账户、多组合隔离。
- 多策略参数化管理。
- 更完整的收益归因和风险暴露报告。

## 运行方式

复制环境变量：

```bash
cp .env.example .env
```

启动服务：

```bash
docker compose up --build
```

访问地址：

- Web 控制台：`http://localhost:5173`
- API 健康检查：`GET http://localhost:8000/health`
- Swagger 文档：`http://localhost:8000/docs`
- ETF 资产列表：`GET http://localhost:8000/api/assets`

## 使用路径

建议按以下顺序使用系统：

1. 在“ETF 池”确认当前启用 ETF。
2. 在“数据健康”同步交易日历和 ETF 行情。
3. 在“因子排名”计算 Alpha 因子。
4. 在“因子研究”验证因子 IC、Rank IC、相关性和分组收益。
5. 在“目标组合”运行策略并生成目标权重。
6. 在“当前持仓”录入已有 ETF 持仓并运行持仓分析。
7. 在“组合工作台”集中查看持仓偏离、定投优先级和调仓动作。
8. 在“定投计划”设置定投周期和每期金额，生成本期投入建议。
9. 在“风控调仓”执行风控检查并生成调仓建议。
10. 在“回测”查看基础历史表现。
11. 在“报告”生成月度复盘报告。
12. 在“全流程”页面把上述核心步骤串成后台任务。

## 后续开发要求

每完成一版必须更新：

如果新增或修改数据库表，必须更新：

- `docs/03_数据库设计.md`
- `sql/init_schema.sql`
- 相关 SQLAlchemy model

如果新增或修改 API，必须更新：

- `docs/04_API接口文档.md`
- 对应 `backend/app/api/` 文件
- 必要的 schema 和 service

如果新增策略、风控、回测、报告、持仓分析或前端页面，必须同步更新对应设计文档和 `docs/10_系统流程说明.md`。
