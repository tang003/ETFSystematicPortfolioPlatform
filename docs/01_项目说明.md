# 01 项目说明

## 项目定位

项目名称：ETF Systematic Portfolio Platform  
中文名称：ETF 系统化资产配置平台

本项目面向个人中长期 ETF 投研与资产配置，目标是把“数据、策略、风控、持仓、调仓、回测、报告”沉淀为可追踪、可复盘、可解释的工程流程。

本系统不是短线预测系统，不是荐股系统，也不是自动实盘交易系统。当前所有交易相关输出都只是建议单，不会连接券商账户，也不会自动操作资金。

## 核心原则

- 数据可信先于回测。
- 风控先于收益。
- 人工确认先于自动交易。
- ETF 优先于个股。
- 中长期配置优先于短线交易。
- 可解释规则优先于黑箱模型。
- 文档与代码同步更新，保证后续可以交给自己或 AI 快速接手。

- 行情清洗和数据质量检查。
- Alpha 因子计算和排名。
- `core_etf_rotation` 策略运行。
- 目标组合生成。
- 独立风控检查。
- 当前持仓快照录入。
- 当前持仓与目标组合差异分析。
- 基于真实当前权重生成调仓建议单。
- 基础回测。
- 月度 Markdown 报告。
- Vue 3 + Element Plus 中文前端控制台。
- 全流程后台任务、任务日志、取消任务、失败步骤重试。

暂未实现能力：

- 自动下单。
- 用户登录和权限管理。
- 多账户、多组合隔离。
- 多策略参数化管理。
- 定投计划自动分析。
- 更完整的收益归因和风险暴露报告。

## 运行方式

复制环境变量：

```bash
cp .env.example .env
```

启动服务：

```bash
docker compose up --build
```

访问地址：

- Web 控制台：`http://localhost:5173`
- API 健康检查：`GET http://localhost:8000/health`
- Swagger 文档：`http://localhost:8000/docs`
- ETF 资产列表：`GET http://localhost:8000/api/assets`

## 使用路径

建议按以下顺序使用系统：

1. 在“ETF 池”确认当前启用 ETF。
2. 在“数据健康”同步交易日历和 ETF 行情。
3. 在“因子排名”计算 Alpha 因子。
4. 在“目标组合”运行策略并生成目标权重。
5. 在“当前持仓”录入已有 ETF 持仓并运行持仓分析。
6. 在“风控调仓”执行风控检查并生成调仓建议。
7. 在“回测”查看基础历史表现。
8. 在“报告”生成月度复盘报告。
9. 在“全流程”页面把上述核心步骤串成后台任务。

每完成一版必须更新：

- `docs/09_开发日志.md`：记录版本、日期、完成内容、修改文件、数据库变化、API 变化、验收标准和后续待办。

如果新增或修改数据库表，必须更新：

- `docs/03_数据库设计.md`
- `sql/init_schema.sql`
- 相关 SQLAlchemy model

如果新增或修改 API，必须更新：

- `docs/04_API接口文档.md`
- 对应 `backend/app/api/` 文件
- 必要的 schema 和 service

如果新增策略、风控、回测、报告、持仓分析或前端页面，必须同步更新对应设计文档和 `docs/10_系统流程说明.md`。
