# 01 项目说明

## 项目定位

项目名称：ETF Systematic Portfolio Platform  
中文名称：ETF 系统化资产配置平台

本项目面向个人中长期 ETF 投研与资产配置，不是短线预测系统，也不是自动实盘交易系统。系统目标是把 ETF 投资过程沉淀为可追踪、可复盘、可解释的工程流程。

核心原则：

- 数据可信先于回测。
- 风控先于收益。
- 人工确认先于自动交易。
- ETF 优先于个股。
- 中长期配置优先于短线交易。
- 可解释规则优先于黑箱模型。

## 第一阶段范围

当前版本为 `v0.1.0-phase1`，只完成基础工程与数据库。

已实现：

- FastAPI 后端项目结构。
- SQLAlchemy 数据库连接配置。
- PostgreSQL 初始化 SQL。
- 需求文档列出的 21 张核心表。
- ETF 初始资产池。
- 默认策略配置和基础风控规则种子数据。
- `GET /health` 健康检查接口。
- `GET /api/assets` ETF 资产列表接口。
- Docker Compose 启动 PostgreSQL、Redis、API 服务。
- 项目维护文档。

暂未实现：

- 行情采集。
- 数据清洗和质量检测执行逻辑。
- 因子计算。
- 策略运行。
- 风控修正。
- 调仓单生成。
- 回测。
- 月度报告生成。
- Vue 前端控制台。
- 真实交易接口。

## 运行方式

复制环境变量：

```bash
cp .env.example .env
```

启动服务：

```bash
docker compose up --build
```

访问：

- `GET http://localhost:8000/health`
- `GET http://localhost:8000/api/assets`
- `http://localhost:8000/docs`

## 后续开发要求

每完成一版必须更新：

- `docs/09_开发日志.md`：记录版本、日期、变更文件、数据库变化、API 变化、待办。

如果新增或修改数据库表，必须更新：

- `docs/03_数据库设计.md`
- `sql/init_schema.sql`
- 相关 SQLAlchemy model。

如果新增或修改 API，必须更新：

- `docs/04_API接口文档.md`

- 必要的 schema 和 service。

如果新增策略、风控、回测逻辑，必须同步更新对应设计文档。
