# 08 部署说明

## 环境要求

- Docker
- Docker Compose
- Python 3.11+

## Docker 启动

在项目根目录执行：

```bash
cp .env.example .env
docker compose up --build
```

服务端口：

- API：`8000`
- Web 控制台：`5173`
- PostgreSQL：宿主机 `5433` 映射到容器 `5432`
- Redis：宿主机 `6380` 映射到容器 `6379`

## 数据库初始化

首次启动 PostgreSQL 容器时会自动执行：

```text
sql/init_schema.sql
sql/seed_assets.sql
```

如果数据库 volume 已存在，初始化 SQL 不会重复执行。需要重新初始化时：

```bash
docker compose down -v
docker compose up --build
```

## 本地后端启动

如果不用 Docker 运行 API：

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

需要确保 `.env` 中 PostgreSQL 地址可访问。

## 本地前端启动

```bash
cd frontend
npm install
npm run dev
```

访问：

```text
http://localhost:5173
```

Vite 开发服务器会把 `/api` 和 `/health` 代理到 `http://localhost:8000`。

前端控制台主要操作入口：

- 全流程：创建后台任务，异步执行同步数据、计算因子、运行策略、风控调仓和生成报告；支持查看历史任务、取消任务和重试失败步骤。
- 数据健康：同步交易日历、同步 ETF 行情、检查缺失数据。
- 因子排名：按日期范围计算 Alpha 因子。
- 目标组合：运行 `core_etf_rotation` 策略并生成目标持仓。
- 当前持仓：录入已有 ETF 持仓，系统按市值计算当前权重，并与目标组合对比生成加仓、减仓、持有或退出建议。
- 定投计划：设置定投期数和每期金额，按目标组合与当前持仓缺口生成本期投入建议。
- 风控调仓：按 `run_id` 执行风控检查并生成调仓单。
- 回测：运行等权买入持有回测并查看净值曲线。
- 报告：生成并查看月度 Markdown 报告。

## 常见问题

### /health 返回 500

优先检查：

- PostgreSQL 容器是否启动。
- `.env` 中 `POSTGRES_HOST` 是否正确。
- 数据库名、用户名、密码是否与 `docker-compose.yml` 一致。

### 修改 SQL 后没有生效

PostgreSQL 初始化脚本只在空数据目录首次执行。开发阶段可以删除 volume 后重建。

### Swagger 打不开

确认 API 容器日志，或本地运行：

```bash
docker compose logs -f api
```

### 行情同步失败

当前行情同步优先使用 AKShare，并在 AKShare 失败时 fallback 到东方财富 K 线 HTTP 接口。若两个来源都不可用，`POST /api/market/sync` 会返回单个 ETF 的失败信息，并写入 `data_quality_log`。可以稍后重试，或缩小到单只 ETF 和较短日期区间验证。

### 缺失数据检查不生效

缺失交易日检查依赖 `trading_calendar`。请先调用：

```bash
POST /api/calendar/sync
```

同步对应日期范围的交易日历后，再执行 `POST /api/data-quality/check`。

### 前端页面没有显示最新内容

如果刚修改了前端代码，但浏览器仍显示旧页面，或控制台提示某个导出不存在，通常是 Vite 开发服务缓存了旧模块。可以重启前端：

```bash
cd frontend
npm run dev
```

然后刷新浏览器页面。

### 持仓分析没有结果

优先检查：

- 是否已经生成目标组合，即 `/api/portfolio/target` 有数据。
- 前端“当前持仓”页的运行 ID 是否对应已有 `strategy_run.id`。
- 是否已经保存当前持仓快照。
- 如果没有持仓快照，调仓单会按当前权重为 0 处理。

### 定投计划没有建议

优先检查：

- 是否已经生成目标组合。
- 定投计划绑定的运行 ID 是否存在。
- 是否已经保存当前持仓快照。
- 如果没有当前持仓，系统会把当前权重按 0 处理，并按目标组合分配本期金额。
