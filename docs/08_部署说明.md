# 08 部署说明

## 本地开发

推荐使用 Docker Compose：

```bash
cp .env.example .env
docker compose up --build
```

本地地址：

- 前端：`http://localhost:5173`
- 后端：`http://localhost:8000`
- Swagger：`http://localhost:8000/docs`

前端单独开发：

```bash
cd frontend
npm install
npm run dev
```

后端单独测试：

```bash
docker compose build api
docker compose run --rm api pytest -q
```

## 环境变量

常用变量：

```bash
POSTGRES_DB=quant_etf
POSTGRES_USER=quant_etf
POSTGRES_PASSWORD=replace-with-secure-password

AUTH_ENABLED=true
AUTH_USERNAME=admin
AUTH_PASSWORD_HASH=replace-with-generated-hash
AUTH_SECRET_KEY=replace-with-generated-secret

TUSHARE_TOKEN=your_token
TUSHARE_REQUEST_INTERVAL_SECONDS=1.5

DEEPSEEK_API_KEY=your_key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-flash
```

生成登录密钥：

```bash
python scripts/generate_auth_secrets.py
```

注意：

- 真实 Key 只写 `.env` 或服务器 `.env.production`。
- 不要提交 Git。
- 已共享到聊天或外部渠道的 Key，正式上线前建议轮换。

## 生产部署

服务器项目目录：

```bash
/opt/ETFSystematicPortfolioPlatform
```

拉取代码并部署：

```bash
cd /opt/ETFSystematicPortfolioPlatform
git pull --ff-only
python3 scripts/deploy_production.py --compose-file compose.production.external-caddy.yml --env-file .env.production
```

部署脚本会自动：

1. 拉取最新代码。
2. 备份 PostgreSQL。
3. 检查 Compose 配置。
4. 构建镜像。
5. 启动容器。
6. 执行 Alembic 迁移。
7. 检查 API ready。

## 数据同步建议

共享 Tushare token 或低积分场景：

- 不要频繁全市场同步。
- 日常同步 `core` 范围。
- ETF 基础池同步当前正式使用 Tushare `fund_basic`；`source=auto` 在正式口径下应优先 Tushare。
- 共享 Tushare token 时，ETF 基础池建议低频执行，例如每天或每周一次，不要连续重复点击。
- 设置 `request_interval_seconds`。
- 限制 `max_symbols`。

建议日常任务：

```bash
python3 scripts/run_core_market_sync.py --scope core --source tushare --max-symbols 30 --interval 1.5
```

ETF 基础池低频同步：

```bash
python3 scripts/run_etf_universe_sync.py --source tushare
```

## 每日自动维护

生产环境可以让 worker 在每天收盘后自动执行一次维护任务。该任务不会下单，也不会操作资金，只更新本地数据和研究结果。

默认关闭：

```bash
DAILY_MAINTENANCE_ENABLED=false
```

推荐在 `.env.production` 中配置：

```bash
DAILY_MAINTENANCE_ENABLED=true
DAILY_MAINTENANCE_TIME=18:30
DAILY_MAINTENANCE_SCOPE=enabled
DAILY_MAINTENANCE_LOOKBACK_DAYS=10
DAILY_MAINTENANCE_MAX_SYMBOLS=120
DAILY_MAINTENANCE_REQUEST_INTERVAL_SECONDS=0.2
DAILY_MAINTENANCE_STRATEGY_CODE=core_etf_rotation
DAILY_MAINTENANCE_PORTFOLIO_VALUE=100000
DAILY_MAINTENANCE_GENERATE_REPORT=true
```

执行顺序：

1. 同步交易日历。
2. 增量同步 ETF 日线行情。
3. 重算启用 ETF 因子。
4. 运行策略并生成目标组合。
5. 执行风控检查。
6. 生成调仓建议单。
7. 可选生成 Markdown 报告。

说明：

- 调度只在 worker 容器中运行，API 容器不注册定时任务。
- 时间按 `Asia/Shanghai` 时区解释。
- Redis 锁会防止同一时间重复执行。
- 如果使用共享 token，建议保留较大的请求间隔，例如 `1.5` 秒；如果是独享或高频额度 token，可以调低。
- 修改 `.env.production` 后需要重新部署或重启 worker 才会生效。

## DBeaver 只读连接

生产 PostgreSQL 默认只映射到服务器本机：

```text
127.0.0.1:5432 -> postgres:5432
```

本地 DBeaver 不应直接连接公网 `5432`，建议使用 SSH 隧道：

- 主机：`127.0.0.1`
- 端口：`5432`
- 数据库：`quant_etf`
- 用户：只读账号，例如 `quant_etf_readonly`
- SSH Host：服务器公网 IP 或 SSH 别名对应 IP
- SSH Port：`22`
- SSH User：服务器登录用户

只读账号仅授予 `SELECT` 权限，适合查看线上数据和排查问题；需要改数据时再临时使用管理员账号。

## 常见问题

### ETF 详情没有数据

通常是该 ETF 在当前日期范围内没有本地清洗行情。先同步该 ETF 或核心池行情。

### 当前持仓提示未找到价格

说明本地没有该代码最新清洗行情。开启自动补全，或手动刷新行情。

### AI 投研没有使用 DeepSeek

检查：

- `DEEPSEEK_API_KEY` 是否配置。
- API 容器是否重启。
- DeepSeek 调用是否超时或失败。

即使 DeepSeek 不可用，系统也会返回规则型多 Agent 结论。
