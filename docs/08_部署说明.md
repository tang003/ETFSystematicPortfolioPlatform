# 08 部署说明

## 本地开发

推荐使用 Docker Compose：

```bash
cp .env.example .env
docker compose up --build
```

本地地址：

- 前端：`http://localhost:5173`
- 后端：`http://localhost:8000`
- Swagger：`http://localhost:8000/docs`

前端单独开发：

```bash
cd frontend
npm install
npm run dev
```

后端单独测试：

```bash
docker compose build api
docker compose run --rm api pytest -q
```

## 环境变量

常用变量：

```bash
POSTGRES_DB=quant_etf
POSTGRES_USER=quant_etf
POSTGRES_PASSWORD=replace-with-secure-password

AUTH_ENABLED=true
AUTH_USERNAME=admin
AUTH_PASSWORD_HASH=replace-with-generated-hash
AUTH_SECRET_KEY=replace-with-generated-secret

TUSHARE_TOKEN=your_token
TUSHARE_REQUEST_INTERVAL_SECONDS=1.5

DEEPSEEK_API_KEY=your_key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-flash
```

生成登录密钥：

```bash
python scripts/generate_auth_secrets.py
```

注意：

- 真实 Key 只写 `.env` 或服务器 `.env.production`。
- 不要提交 Git。
- 已共享到聊天或外部渠道的 Key，正式上线前建议轮换。

## 生产部署

服务器项目目录：

```bash
/opt/ETFSystematicPortfolioPlatform
```

拉取代码并部署：

```bash
cd /opt/ETFSystematicPortfolioPlatform
git pull --ff-only
python3 scripts/deploy_production.py --compose-file compose.production.external-caddy.yml --env-file .env.production
```

部署脚本会自动：

1. 拉取最新代码。
2. 备份 PostgreSQL。
3. 检查 Compose 配置。
4. 构建镜像。
5. 启动容器。
6. 执行 Alembic 迁移。
7. 检查 API ready。

## 常用检查命令

查看容器：

```bash
docker compose --env-file .env.production -f compose.production.external-caddy.yml ps
```

健康检查：

```bash
curl https://etf.8886767.xyz/health/ready
```

检查 DeepSeek 是否配置：

```bash
docker compose --env-file .env.production -f compose.production.external-caddy.yml exec -T api python - <<'PY'
from app.core.config import get_settings
s = get_settings()
print({"deepseek_configured": bool(s.deepseek_api_key), "model": s.deepseek_model})
PY
```

检查数据库表：

```bash
docker compose --env-file .env.production -f compose.production.external-caddy.yml exec -T api python - <<'PY'
from sqlalchemy import inspect
from app.core.database import engine
print(inspect(engine).get_table_names())
PY
```

## 数据同步建议

共享 Tushare token 或低积分场景：

- 不要频繁全市场同步。
- 日常同步 `core` 范围。
- 设置 `request_interval_seconds`。
- 限制 `max_symbols`。

建议日常任务：

```bash
python3 scripts/run_core_market_sync.py --scope core --source tushare --max-symbols 30 --interval 1.5
```

ETF 基础池低频同步：

```bash
python3 scripts/run_etf_universe_sync.py --source akshare
```

## 常见问题

### ETF 详情没有数据

通常是该 ETF 在当前日期范围内没有本地清洗行情。先同步该 ETF 或核心池行情。

### 当前持仓提示未找到价格

说明本地没有该代码最新清洗行情。开启自动补全，或手动刷新行情。

### AI 投研没有使用 DeepSeek

检查：

- `DEEPSEEK_API_KEY` 是否配置。
- API 容器是否重启。
- DeepSeek 调用是否超时或失败。

即使 DeepSeek 不可用，系统也会返回规则型多 Agent 结论。
