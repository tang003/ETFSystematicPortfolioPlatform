# 08 部署说明

## 环境要求

- Docker
- Docker Compose
- Python 3.11+

## Docker 启动

在项目根目录执行：

```bash
cp .env.example .env
docker compose up --build
```

服务端口：

- API：`8000`
- PostgreSQL：宿主机 `5433` 映射到容器 `5432`
- Redis：宿主机 `6380` 映射到容器 `6379`

## 数据库初始化

首次启动 PostgreSQL 容器时会自动执行：

```text
sql/init_schema.sql
sql/seed_assets.sql
```

如果数据库 volume 已存在，初始化 SQL 不会重复执行。需要重新初始化时：

```bash
docker compose down -v
docker compose up --build
```

## 本地后端启动

如果不用 Docker 运行 API：

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

需要确保 `.env` 中 PostgreSQL 地址可访问。

## 常见问题

### /health 返回 500

优先检查：

- PostgreSQL 容器是否启动。
- `.env` 中 `POSTGRES_HOST` 是否正确。
- 数据库名、用户名、密码是否与 `docker-compose.yml` 一致。

### 修改 SQL 后没有生效

PostgreSQL 初始化脚本只在空数据目录首次执行。开发阶段可以删除 volume 后重建。

### Swagger 打不开

确认 API 容器日志，或本地运行：

```bash
docker compose logs -f api
```

### 行情同步失败

阶段 2 使用 AKShare 拉取 ETF 行情，并在 AKShare 失败时 fallback 到东方财富 K 线 HTTP 接口。若两个来源都不可用，`POST /api/market/sync` 会返回单个 ETF 的失败信息，并写入 `data_quality_log`。可以稍后重试，或缩小到单只 ETF 和较短日期区间验证。

### 缺失数据检查不生效

缺失交易日检查依赖 `trading_calendar`。请先调用：

```bash
POST /api/calendar/sync
```

同步对应日期范围的交易日历后，再执行 `POST /api/data-quality/check`。
