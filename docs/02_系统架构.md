# 02 系统架构

## 当前架构

```text
ETFSystematicPortfolioPlatform
├── backend
│   └── app
│       ├── api          # FastAPI 路由
│       ├── core         # 配置、数据库、日志、调度器
│       ├── models       # SQLAlchemy ORM
│       ├── schemas      # Pydantic 响应和请求模型
│       └── services     # 业务服务
├── sql                  # 初始化建表和种子数据
├── docs                 # 设计文档和开发日志
└── docker-compose.yml   # PostgreSQL、Redis、API 编排
```

## 模块职责

### backend/app/core

- `config.py`：统一读取 `.env` 环境变量，生成数据库连接串。
- `database.py`：创建 SQLAlchemy engine、session 和 `get_db` 依赖。
- `logging.py`：初始化 loguru 日志。
- `scheduler.py`：预留 APScheduler 定时任务入口。

### backend/app/models

每张核心业务表对应一个 SQLAlchemy model。阶段 1 只完成表结构映射，不实现复杂业务逻辑。

### backend/app/api

当前路由：

- `health_api.py`：健康检查。
- `asset_api.py`：ETF 主数据查询。
- `calendar_api.py`：交易日历同步和查询。
- `market_api.py`：行情同步、raw/clean 行情查询。
- `data_quality_api.py`：数据质量检查、日志和状态查询。

后续新增行情、因子、策略、风控、回测、报告接口时继续按领域拆分文件。

### backend/app/services

业务逻辑层。API 不直接堆业务规则，复杂逻辑应放入 service，方便测试和复用。

## 数据流

阶段 1 仅完成基础数据结构：

```text
seed_assets.sql
→ asset_master / strategy_config / risk_rule
→ SQLAlchemy service
→ FastAPI API
→ 用户或前端查询
```

阶段 2 新增行情数据闭环：

```text
AKShare
→ POST /api/market/sync
→ AKShare 失败时 fallback 到东方财富 K 线 HTTP 接口
→ market_data_raw
→ market_data_clean
→ data_quality_log
→ GET /api/market/bars/{symbol}
```

交易日历流：

```text
AKShare 交易日历
→ POST /api/calendar/sync
→ trading_calendar
→ 数据质量检查判断 open day 缺失
```

未来完整数据流：

```text
外部行情源
→ market_data_raw
→ 数据清洗
→ market_data_clean
→ 数据质量日志
→ factor_daily
→ alpha_signal
→ target_portfolio
→ risk_check_result
→ rebalance_order
→ report_log
```

## 策略流

后续策略引擎应遵循：

```text
读取 strategy_config
→ 创建 strategy_run
→ 读取 factor_daily
→ 生成 alpha_signal
→ 生成 target_portfolio.raw_target_weight
→ 交给风控引擎修正
→ 保存 final_target_weight
```

## 风控流

风控必须独立于策略：

```text
策略原始仓位
→ 风控规则 risk_rule
→ 风控检查 risk_check_result
→ 风控后目标仓位
→ 调仓单
```

## 回测流

后续回测模块应输出：

- `backtest_run`
- `backtest_equity_curve`
- `backtest_trade`
- `backtest_metrics`

## 报告流

后续报告模块应把 Markdown 报告落到：

- `report_log.content_markdown`
- 可选文件路径 `report_log.file_path`
