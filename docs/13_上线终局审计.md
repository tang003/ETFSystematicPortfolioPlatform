# 13 上线终局审计

本文用于把系统按正式 ETF 量化平台的模块逐项审核，方便后续继续交给 AI 或人工开发时快速定位优先级。

## 当前结论

系统已经具备 ETF 量化平台的主干能力：ETF 主数据、历史行情、数据健康、因子、筛选、对比、策略、回测、目标组合、当前持仓、定投、风控调仓、报告、AI 投研、生产部署和只读数据库连接。

上线前仍需重点补齐三类能力：

- 数据维度：规模/份额、折溢价、跟踪误差、成分和行业暴露仍不完整。
- 组合约束：组合层面的相关性、行业/主题集中度、单资产上限需要更强约束。
- 运营能力：日志清理、任务告警、备份演练、用户权限分层和审计记录还需要强化。

## 模块审计

| 模块 | 当前状态 | 主要不足 | 上线优先级 |
|---|---|---|---|
| 总览 | 可展示核心指标和近期数据 | 缺少一眼可见的“今日待处理事项”和数据新鲜度 | P1 |
| ETF 池 | 支持全市场基础池、精选池、启用研究 | 缺少批量标签、同指数分组、规模/费率完整度提示 | P1 |
| ETF 筛选 | 已支持买入评分排行 | 下一步应加入估值、规模、折溢价、跟踪误差过滤 | P0 |
| ETF 对比 | 已支持收益、风险、交易性、相关性 | 仍缺同指数费率/规模/跟踪误差深度对比 | P1 |
| ETF 详情 | 已有行情、风险、替代 ETF | 缺少成分暴露、行业暴露、历史份额变化 | P1 |
| 数据健康 | 已支持范围同步、质量日志、快捷日期 | 质量日志增长较快，需要保留策略和清理任务 | P0 |
| 因子排名 | 已有趋势、动量、波动、回撤、流动性 | 缺少因子稳定性、拥挤度、因子失效提醒 | P1 |
| 因子研究 | 已有基础研究 | 缺少样本外验证、滚动 IC、分组净值 | P2 |
| 策略/目标组合 | 已能生成目标组合 | 需要组合约束、风险预算、相关性惩罚、主题集中度限制 | P0 |
| 当前持仓 | 支持录入、补全、分析 | 需要自动启用持仓 ETF、持仓与目标组合差异更直观 | P0 |
| 定投计划 | 支持周期金额和建议 | 需要目标收益约束、现金管理和买入节奏解释更清楚 | P1 |
| 风控调仓 | 已有调仓建议 | 需要止损/止盈/再平衡阈值可配置 | P1 |
| 回测 | 已能跑策略回测 | 需要交易成本敏感性、样本外、滚动回测、回撤恢复期 | P1 |
| 报告 | 可生成报告 | 需要面向普通用户的中文摘要和风险声明模板 | P1 |
| AI 投研 | 已接 DeepSeek 多 Agent | 需要引用本地指标、新闻/公告数据源和结论可追溯 | P1 |
| 系统状态 | 有健康检查 | 需要数据源状态、token 频率、任务队列、磁盘水位展示 | P0 |
| 部署安全 | 已有鉴权、HTTPS、只读 DB | 需要用户权限分层、操作审计、定期恢复演练 | P0 |

## 上线前 P0 清单

- 数据健康日志清理机制。
- ETF 筛选页稳定可用，作为普通用户第一入口。
- 当前持仓新增后自动启用并补行情。
- 目标组合生成时加入相关性和单主题集中度约束。
- 系统状态页展示数据库大小、worker 心跳、数据源 token 状态。
- 生产备份恢复演练至少完成一次。

## 已在 v0.51 补强

- 新增 ETF 筛选页和 `/api/etf-compare/screener` 接口。
- 筛选器支持按启用池、核心池、持仓、目标、定投、全资产和手动代码筛选。
- 筛选排序使用买入评分、可交易性、夏普和年化收益。
