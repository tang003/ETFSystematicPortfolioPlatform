# 00 文档导航

本文档是 ETF Systematic Portfolio Platform 的维护入口。以后自己维护项目，或把项目交给 AI 修改时，建议先提供本文件、`README.md`、`docs/10_系统流程说明.md` 和本次任务相关专题文档。

## 推荐阅读顺序

### 快速理解项目

1. `README.md`：项目定位、当前版本、启动方式、常用地址。
2. `AGENTS.md`：给 Codex / AI 继续开发项目的总规范。
3. `docs/01_项目说明.md`：业务边界、当前能力、未完成能力。
4. `docs/10_系统流程说明.md`：从用户操作和系统流程理解平台如何运行。
5. `docs/15_分析周期口径说明.md`：理解各页面“分析周期”和未来定投周期的区别。
6. `docs/16_策略与数据源管理说明.md`：理解策略新增、策略参数、Tushare/DeepSeek 配置和 Token 安全边界。
7. `docs/09_开发日志.md`：查看每个版本做了什么。

### 准备修改后端

1. `docs/02_系统架构.md`：理解后端、前端、数据库、任务执行的分层。
2. `docs/03_数据库设计.md`：确认表结构、字段含义和迁移约定。
3. `docs/04_API接口文档.md`：确认接口路径、请求参数和响应结构。
4. 根据任务阅读专题文档，例如策略、风控、回测、部署。

### 准备修改前端

1. `frontend/src/router/index.ts`：页面路由。
2. `frontend/src/api/client.ts`：前端 API 客户端和类型定义。
3. `frontend/src/views`：业务页面。
4. `frontend/src/styles.css`：全局样式。

### 准备部署或排查

1. `docs/08_部署说明.md`
2. `docs/11_服务器上线清单.md`
3. `scripts/deploy_production.py`
4. `compose.production.external-caddy.yml`

## 版本维护规则

每完成一个可验收版本，需要同步做四件事：

1. 更新相关设计文档。
2. 在 `docs/09_开发日志.md` 顶部增加版本记录。
3. 跑必要测试和构建。
4. 提交 Git 并推送 GitHub。

建议版本命名：

```text
v主版本.次版本.修订号-阶段说明
```

示例：

```text
v0.39.0-agent-analysis-history
v0.40.0-docs-refresh
```

## 当前关键边界

- 当前只做 ETF，不做完整股票交易平台。
- 当前不连接真实券商账户。
- 当前不自动下单。
- 所有买卖、调仓、定投建议都需要人工确认。
- DeepSeek 只负责总结和解释，不负责凭空生成行情。
- 当前正式 ETF 数据源以 Tushare 为主；AKShare / Eastmoney 旧代码和历史日志只作追溯，不应作为默认业务入口。
- 外部数据源都可能失败，系统需要保留本地数据、错误提示、重试和后台任务日志。
- 用户持仓目前推荐手动录入，后续如接券商只读同步，需要单独做权限、隐私和合规设计。
