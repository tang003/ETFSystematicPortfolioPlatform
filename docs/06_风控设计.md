# 06 风控设计

当前已实现基础独立风控引擎、当前持仓读取和调仓单生成。风控规则保存在 `risk_rule`，检查结果保存在 `risk_check_result`，调仓建议保存在 `rebalance_order`。

## 已初始化规则

- 单一 ETF 最大仓位：`max_single_weight <= 50%`
- 权益资产最大仓位：`max_equity_weight <= 80%`
- 防御资产最低仓位：`min_defense_weight >= 20%`
- 现金最低仓位：`min_cash_weight >= 5%`
- 跨境 ETF 新增买入溢价限制：`premium_buy_limit <= 3%`

## 后续实现要求

风控引擎必须独立于策略引擎：

```text
策略生成 raw_target_weight
→ 风控读取 risk_rule
→ 保存 risk_check_result
→ 输出 final_target_weight
```

所有风控修正必须保存原因，不能只覆盖结果。

## 已实现风控流程

```text
target_portfolio.raw_target_weight
→ run_risk_check
→ risk_check_result
→ target_portfolio.final_target_weight
→ portfolio_position 当前持仓权重
→ generate_rebalance_orders
→ rebalance_order
```

## 已实现规则

### 单一 ETF 最大仓位

规则代码：`max_single_weight`

如果单只 ETF 权重大于阈值，超出部分转入现金资产。

### 权益资产最大仓位

规则代码：`max_equity_weight`

如果权益资产总仓位超过阈值，按比例压降权益资产，释放仓位转入现金资产。

### 防御资产最低仓位

规则代码：`min_defense_weight`

如果债券/黄金等防御资产低于阈值，按比例从非防御资产中挪出仓位补足。

### 现金最低仓位

规则代码：`min_cash_weight`

如果现金资产低于阈值，按比例从非现金资产中挪出仓位补足。

## 调仓单生成

当前版本调仓单基于风控后的 `final_target_weight` 和当前持仓权重生成。

如果已经录入 `portfolio_position`，系统会读取最新持仓快照中的真实当前权重：

```text
current_weight = portfolio_position.weight
```

如果尚未录入当前持仓，则默认：

```text
current_weight = 0
```

估算金额：

```text
estimated_amount = portfolio_value * (target_weight - current_weight)
```

最小调仓阈值：

```text
abs(weight_diff) < 5% → HOLD
```
