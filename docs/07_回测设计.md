# 07 回测设计

当前已实现基础等权买入持有回测和策略月度调仓回测。等权买入持有用于基线验证；策略月度调仓用于验证 Alpha 排名、目标组合和月度追加资金组合后的效果。

## 相关表

- `backtest_run`：回测任务。
- `backtest_equity_curve`：净值曲线。
- `backtest_trade`：交易记录。
- `backtest_metrics`：绩效指标。

## 后续默认参数

```text
initial_cash = 10000
monthly_contribution = 2000
fee_rate = 0.001
slippage_rate = 0.001
rebalance_frequency = monthly
```

## 已实现回测模式

### equal_weight_buy_and_hold

流程：

```text
读取 market_data_clean
→ 选择 symbols
→ 首日等权买入
→ 按每日 close 计算组合市值
→ 记录 backtest_equity_curve
→ 记录初始交易 backtest_trade
→ 计算 backtest_metrics
```

当前支持参数：

- `symbols`
- `start_date`
- `end_date`
- `initial_cash`
- `monthly_contribution`
- `fee_rate`
- `slippage_rate`

当前已实现指标：

- `cumulative_return`
- `annualized_return`
- `max_drawdown`
- `volatility`
- `sharpe_ratio`
- `trade_count`

### core_etf_rotation_monthly

流程：

```text
读取 market_data_clean
→ 读取 factor_daily
→ 每月第一个可交易日读取最近 Alpha 排名
→ 复用策略目标权重构造规则
→ 按目标权重卖出高配资产、买入低配资产
→ 月份切换时加入 monthly_contribution
→ 记录 backtest_trade
→ 记录 backtest_equity_curve
→ 计算 backtest_metrics
```

当前限制：

- 回测内使用的是简化风控前目标权重构造，尚未完整复用独立风控引擎。
- 如果某些 ETF 在回测区间缺少行情，会使用前值填充；完全没有行情的日期会跳过。
- 指标中的累计收益率会使用 `initial_cash + monthly_contribution` 形成的总投入本金作为分母。
