# 07 回测设计

## 回测目标

回测模块用于验证策略、定投计划或买入持有组合在历史区间内的表现。当前回测用于研究和复盘，不代表未来收益。

## 当前支持

- 买入持有回测。
- 月度轮动策略回测。
- 月度追加资金。
- 交易记录。
- 净值曲线。
- 基础指标。

## 策略引擎复用

月度轮动回测已经接入内部策略引擎注册制。

当前流程：

```text
历史清洗行情 market_data_clean
 -> 历史因子 factor_daily
 -> 按日期构造 StrategyRunContext
 -> 通过 registry 获取 factor_rotation 引擎
 -> engine.build_targets 生成当期目标权重
 -> 回测撮合买入/卖出并保存交易和净值
```

这样可以减少“策略运行”和“回测运行”两套口径分叉。后续新增算法型策略时，应优先让回测也通过同一个策略引擎生成目标权重。

## 主要数据表

- `backtest_run`：回测任务。
- `backtest_equity_curve`：每日净值曲线。
- `backtest_trade`：模拟交易记录。
- `backtest_metrics`：回测指标。

## 输入参数

常见参数：

- `strategy_code`
- `name`
- `symbols`
- `start_date`
- `end_date`
- `initial_cash`
- `monthly_contribution`
- `fee_rate`
- `slippage_rate`

## 输出指标

当前或建议输出：

- 总收益。
- 年化收益。
- 年化波动。
- 最大回撤。
- 胜率。
- 回测期数。
- 交易次数。

## 当前限制

- 交易成本和滑点是简化模型。
- 未接入真实分红、拆分、复权全细节。
- ETF 溢价、申赎、停牌等特殊情况处理仍需增强。
- 回测不能代表未来收益。
- 当前月度轮动回测复用策略引擎生成目标权重，但可交易性评分在历史切片中暂未逐日重放，后续需要加入历史交易性上下文。

## 后续优化

- 增加基准对比。
- 增加月度收益表。
- 增加夏普、卡玛、索提诺。
- 增加等权组合、目标风险组合对比。
- 把更多策略引擎接入回测统一入口。
- 把历史可交易性、折溢价和跟踪误差作为回测时点上下文。
- 把 AI 投研结论纳入回测复盘说明，但不让 AI 改写历史交易结果。
